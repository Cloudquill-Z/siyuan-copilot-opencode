# 第二轮全量评审修复设计

## 目标

修复第二轮项目级审查确认的数据可靠性、状态竞态、配置归一化和安装流程问题，同时保持现有 UI 与 OpenCode 传输架构不变。

## 设计

- 导出名称继续由 `sessionExport` 统一解析，但平台名始终以消息保存的完整模型 ID 为准，当前配置只刷新展示名称。
- `putFile` 与通用 SiYuan API 一样采用失败即抛错；会话保存捕获错误后保留未保存状态，不再发送成功信号。
- 健康轮询持有当前请求控制器和运行代次；切换服务器或停止轮询会中止旧请求，旧代次不得写回状态。
- 设置归一化使用有限数值判断，允许合法的 0；长期记忆只有模型明确返回 `shouldRemember: true` 才允许写入。
- 安装脚本读取当前运行内核的 `workspaceDir`，多工作空间时优先自动选择当前工作空间；非交互环境无法确认目标时明确跳过，不再假装安装成功。
- `@mozilla/readability` 升级到修复 ReDoS 的 0.6.x，并通过现有网页解析构建链验证兼容性。

## 验证

新增或扩充 Node 行为测试覆盖模型平台解析、零值设置、严格记忆开关、文件写入失败、健康轮询失效保护和工作空间选择。完成全部行为测试、TypeScript 检查、生产构建、目标目录版本核验和 graphify 更新后提交并推送。
