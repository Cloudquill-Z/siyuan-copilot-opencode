# 设置页信息架构重构设计

## 目标

将设置页从按代码模块平铺的八个同级入口，重组为按用户任务组织的五个一级页面。保留现有功能与思源原生视觉语言，降低寻找设置、理解依赖和处理危险操作的成本。

## 信息架构

1. **连接与模型**：OpenCode 服务地址、连接状态、服务操作和模型管理。
2. **对话体验**：用户名称、发送快捷键、执行中发送、最大并行任务数、消息字号和系统提示词。
3. **自动化与导出**：会话自动命名、重命名模型与提示词、保存笔记本和默认路径。
4. **长期记忆**：总开关、记忆笔记本、根目录、核心档案、提取策略和注入限制。
5. **诊断与维护**：Token 统计、诊断日志和危险区域中的恢复默认设置。

## 交互原则

- 页面内部按照“总开关或状态 → 常用配置 → 依赖配置 → 高级配置”排序。
- 自动重命名关闭时隐藏模型和提示词；长期记忆关闭时保留入口说明，但将后续配置折叠为禁用状态提示。
- 诊断日志关闭时隐藏日志详细程度。
- Token 统计作为诊断页的只读信息，不再占用一级导航。
- 恢复默认设置位于诊断页底部的危险区域，不再占用一级导航。
- 自动保存状态继续全局可见；保存失败和文档校验结果继续就地反馈。

## 数据与兼容性

不修改设置字段、默认值和持久化结构，只调整 `SettingsPanel.svelte` 中的展示分组及条件渲染。已有用户设置无需迁移。

## 验证

- 回归测试校验五个一级入口及旧入口消失。
- 回归测试校验系统提示词、自动命名、导出、记忆、诊断和重置均归入新页面。
- 运行现有回归测试、工具测试和生产构建。
- 安装到思源后检查五页导航、条件展开和暗色模式布局。
