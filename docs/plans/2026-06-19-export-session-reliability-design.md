# 会话导出与持久化可靠性设计

## 目标

统一聊天复制、保存到思源笔记和 Markdown 文件下载的格式化规则，修复名称过期、历史会话错存、空文件、重复多模型回答、错误成功提示及跨页签元数据覆盖问题。

## 架构

新增纯 TypeScript 会话导出模块，负责角色名称解析、当前模型目录名称解析、消息筛选和 Markdown 生成。UI 组件只负责收集设置与消息快照，不再自行拼接 Markdown。用户名始终读取当前设置；助手名称保留消息原始 provider/modelId，并从当前模型配置刷新显示名称，缺失元数据时才回退当前模型。

保存对话框持有独立的导出快照（消息、会话标题和会话 ID），历史会话打开对话框后不再依赖全局 `messages`。SessionManager 的文件下载改为把 sessionId 交给父组件，由父组件读取独立会话文件后生成下载内容。

SiYuan API 请求在非零响应码时抛出带消息的异常，调用方只有拿到有效文档 ID 才报告成功。会话元数据更新通过模块级异步队列串行化同一页面中的多个侧栏实例，并在队列中重新读取和合并最新列表。

## 数据约束

- `modelName` 仅保存纯名称，选中状态只由 `isSelected` 表示。
- 导出仅包含 user/assistant 消息，不把 tool/system 消息伪装成助手。
- 多模型消息只输出一次“多模型对比”，不在其前面重复输出选中回答。
- 文档名称移除路径分隔符，避免会话标题意外改变保存目录。

## 错误处理与生命周期

API 失败向上传播并保留内核错误消息。延时搜索和全局点击监听在组件销毁时统一清理，避免销毁后的回调重新注册监听器。

## 验证

使用可直接由 Node 执行的纯函数回归测试覆盖导出名称、消息过滤、多模型去重、快照和模型名清理；保留现有行为测试，修正已经失效的上下文默认值断言。最终执行全部行为测试、TypeScript 检查、生产构建和 graphify 增量更新。
