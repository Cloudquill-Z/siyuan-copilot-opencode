# 侧栏恢复与任务耗时统计设计

## 目标

修复 SiYuan 重启后 Dock 显示为已打开但 AI 侧栏内容为空的问题，并为每个发出的任务增加实时耗时显示。

## 设计

- Dock 恢复不再只依赖按钮 active 或面板可见性，而是检查真实 Dock model、DOM 是否已连接、组件是否已经挂载，以及面板内容是否缺少插件根节点。
- 将侧栏挂载封装为幂等函数：同一元素已挂载时跳过；元素变化时由注册表销毁旧实例并重新挂载。
- `AISidebar` 根节点增加稳定标记，让恢复逻辑能识别“面板存在但内容为空”的状态。
- 任务运行态增加 `startedAt` 和 `finishedAt`。任务发出前记录开始时间，完成、失败或中断时记录结束时间。
- UI 每秒刷新一次当前时间，只保存时间戳，不为每个后台任务创建计时器；切换任务时根据运行态时间戳计算显示值。

## 验证

- 增加回归检查，覆盖空 Dock 面板必须挂载、已挂载面板必须停止、耗时格式化、任务开始/结束时间流转。
- 执行工具测试、回归测试、生产构建。
- 最终同步提升 patch 版本并通过构建流程安装插件。
