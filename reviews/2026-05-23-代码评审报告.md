### 🔴 高严重度 (9 项)

| # | 文件 | 行号 | 问题描述 |
|---|------|------|----------|
| 1 | `src/index.ts` | 1772-1846 | **全局 click 监听器永不移除** — `setupLinkClickListener` 添加的 `document.addEventListener('click', ..., true)` 在插件卸载或设置变更时都不会被移除。即使用户关闭 `openLinksInWebView` 设置，拦截行为仍然生效，且 `onunload()` 中未清理此监听器。 |
| 2 | `src/index.ts` | 2144-2289 | **双重迁移可能产生重复 customProviders** — `loadSettings()` 在 `onload()`（459 行）和 `onLayoutReady()`（1703 行）各调用一次。V3 迁移逻辑（2151-2184 行）两次执行且操作的是本地对象副本，可能导致重复的 provider 条目。 |
| 3 | `src/ai-chat.ts` | 179 | **模型缓存完全失效** — `fetchModels` 每次调用前都无条件执行 `invalidateModelCache(serverUrl)`，使得 provider 端的缓存机制完全失效，每次都发起网络请求。 |
| 4 | `src/ai-chat.ts` | 225-244 | **ChatOptions 参数静默丢弃** — `temperature`, `maxTokens`, `stream`, `thinkingBudget`, `onToolCallComplete`, `enableImageGeneration`, `onImageGenerated` 这些字段被调用方传入但从未读取或转发给 OpenCode provider。API 契约与实际行为不一致。 |
| 5 | `src/ai-sidebar.svelte` | 634 | **工具选择逻辑反转** — `chatMode === 'plan'` 时使用 `selectedToolsAsk`（问询模式工具），反之使用 `selectedTools`。变量名与逻辑含义相反，导致错误模式下工具集被误用。 |
| 6 | `src/ai-sidebar.svelte` | 1042 | **userToolCount 硬编码为 0** — `$: userToolCount = 0;` 始终赋值为 0，导致所有依赖工具计数的条件分支（如 471-482 行的 `plan` 模式逻辑）永远为假，整个 agent/工具功能被实质性禁用。 |
| 7 | `src/ai-sidebar.svelte` | 2205 | **无效的 chatMode 默认值** — 默认值 `'ask'` 不在 `ChatMode` 类型 (`'plan' \| 'build'`) 中。当 `newSettings.chatMode` 为 `undefined` 时导致运行时类型错误。 |
| 8 | `src/providers/opencode-provider.ts` | 854-891 | **SSE 流错误被静默吞没** — `connect()` 中的 async IIFE 未被 await，返回的 Promise 在 fetch 成功后立即 resolve，流的读取循环错误被 catch 并 log 后静默丢弃，调用方无法感知连接已死亡。 |
| 9 | `scripts/make_dev_copy.js` | 13 | **硬编码 Windows 路径使脚本不可用** — `targetDir = 'D:\\Notes\\Siyuan\\Achuan-2\\data\\plugins'` 被硬编码，导致其后的自动检测 fallback 逻辑（21-40 行）全部不可达。其他机器上脚本必然失败。 |

## 三、类型安全问题

### `any` 类型泛滥

`any` 在代码库中被广泛使用，主要集中在以下模式：

| 模式 | 典型文件 | 影响 |
|------|----------|------|
| Plugin 对象 | `ai-sidebar.svelte:84`, `SettingsPanel.svelte:14`, `WebAppManager.svelte:6` | 所有 `plugin.saveSettings()`, `plugin.loadSettings()` 等调用均无类型检查 |
| Settings/Config 对象 | `ai-sidebar.svelte:112`, `stores/settings.ts:5,8` | 属性名拼写错误只能在运行时发现 |
| Event 参数 | `index.ts:1263,1280,1325...` | webview 事件属性访问无类型安全 |
| webview 类型 | `index.ts:984` | `as any` 使 `canGoBack()`, `goBack()` 等调用均未类型化 |
| SSE 解析函数 | `opencode-provider.ts: ~50 处` | 所有 `part`, `props`, `delta` 参数均为 `any` |

### 补充类型问题

| # | 文件 | 问题 |
|---|------|------|
| 47 | `src/ai-chat.ts:162` | `sessionCleanupQueue.shift()!` 非空断言虽安全但绕过类型检查器 |
| 48 | `src/ai-chat.ts:330,338` | `executeCommand` 返回类型 `CommandResult` 中 drop 了 `info` 字段 |
| 49 | `src/components/ChatDialog.svelte:9` | `AISidebar` 是组件类，用作实例类型不正确 |
| 50 | `src/components/ProviderConfigPanel.svelte:124, 161` | catch 变量未类型收窄，`e.message` 不安全访问 |

| 52 | `src/stores/connectionStatus.ts:26-30` | 手动 subscribe/unsubscribe 模式应使用 `get()` |
| 53 | `src/defaultSettings.ts` | 大量 `as` 断言掩盖结构不匹配 |
